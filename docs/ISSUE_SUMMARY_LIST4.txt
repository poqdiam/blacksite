ITEM 1 — SSP Analyzer download report returns 500  [BLOCKER] — DONE
──────────────────────────────────────────────────────────────
Status: DONE
Root cause: Two-layer failure
  Layer 1 — Python 3.8 / reportlab 4.4.3 incompatibility:
    reportlab 4.4.3 calls md5(usedforsecurity=False), a Python 3.9+ keyword.
    Server runs Python 3.8.  Every invocation of _generate_ssp_pdf() raised:
      TypeError: 'usedforsecurity' is an invalid keyword argument for md5()
  Layer 2 — Missing error template:
    The except block in admin_ssp_pdf tried to render "error_pdf.html",
    which did not exist, causing a secondary TemplateNotFound exception
    that bypassed the route's own handler and fell to the global 500 handler.

Fix applied:
  1. Downgraded reportlab: 4.4.3 → 3.6.12 (Python 3.8 compatible)
       pip install "reportlab==3.6.12"
  2. Created templates/error_pdf.html — themed error page with:
       - Event reference displayed as SSP-{req_id}
       - Truncated error message (no stack trace shown to user)
       - Retry Download / Back to Review / SSP Analyzer buttons
       - Diagnostic block: route, timestamp, event ref
  3. Updated admin_ssp_pdf route (app/main.py ~line 7843) to:
       - Wrap _generate_ssp_pdf() call in try/except
       - On exception: generate req_id, log review_id / user / system_name /
         findings_count / analysis_json_bytes / full traceback
       - Return TemplateResponse("error_pdf.html", {..., "req_id": req_id}, 500)
  4. Added hx-boost="false" to PDF download link in admin_ssp_review.html
     so HTMX does not intercept the binary download response
  5. Added error state block in admin_ssp_review.html for rev.status == "error"

Verification:
  curl -s -o /tmp/test.pdf -w "%{http_code} %{size_download}" \
    -H "Remote-User: dan" http://127.0.0.1:8100/admin/ssp/review/1/pdf
  Result: 200  76652
  file /tmp/test.pdf → PDF document, version 1.4 ✓

Files changed:
  app/main.py            (PDF route, logging, error handler)
  templates/error_pdf.html         (NEW)
  templates/admin_ssp_review.html  (hx-boost=false, error state)
  requirements.txt       (reportlab pinned to 3.6.12)

ITEM 4 — Sidebar and alert ticker persist across navigation  [HIGH] — DONE
──────────────────────────────────────────────────────────────
Status: DONE

Root cause:
  - initSbAdvisory() and initSbQuiz() were bound to DOMContentLoaded (fires once)
    which is correct; however no guard prevented re-invocation from code paths
    that could call them repeatedly.
  - Sidebar advisory IS outside the HTMX #main-content swap target — it persists
    across navigation at the DOM level, but the advisory fetch was not TTL-cached,
    meaning each time code was called it would re-fetch and re-animate.
  - Sidebar active link was server-rendered in <aside> (not in swap target), so it
    did not update as the user navigated without a full page reload.
  - Observation badge fetch was not re-run after HTMX partial swaps.

Fix applied:
  1. initSbQuiz(): added _bsvQuizReady flag — function returns immediately if
     already initialised, preventing double-init on any code path.
  2. initSbAdvisory(): added 15-minute TTL cache:
       var _bsvAdvFetchedAt = 0;
       var _bsvAdvCached    = null;
     If within TTL and cache populated, skips fetch and re-population entirely.
     Inner element gets data-adv-loaded attribute so even forced re-calls skip
     re-injection of already-rendered HTML.
  3. _updateSbActiveLink(): new function that reads window.location.pathname and
     toggles .active class + cyan outline on .sb-link anchors whose href matches.
  4. htmx:afterSettle listener:
       - Calls _updateSbActiveLink() on every partial swap
       - Re-runs observation badge fetch so count stays current across navigation

Verification:
  - Navigate dashboard → systems → /poam → /advisories via HTMX links
  - Advisory ticker does not re-fetch or re-animate within 15-minute window
  - Active sidebar link updates to match current route on each navigation
  - Observation badge count refreshes on each navigation without full page reload

Files changed:
  templates/base.html    (_bsvQuizReady guard, TTL cache vars, _updateSbActiveLink,
                          htmx:afterSettle listener)
