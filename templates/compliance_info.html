Platform Compliance & Security

Information about BLACKSITE's security controls, accessibility conformance, and supported compliance frameworks.

{# ── 1. Accessibility ─────────────────────────────────────────────────────── #}

1. Accessibility

BLACKSITE targets WCAG 2.1 Level AA partial conformance. Accessibility improvements implemented include: skip navigation, ARIA labels on all interactive controls, modal focus management, semantic table markup, live regions for dynamic content, and keyboard operability throughout the platform.

View Accessibility Statement  ·  View VPAT (Voluntary Product Accessibility Template)

{# ── 2. Section 508 ───────────────────────────────────────────────────────── #}

2. Section 508 (2018 Refresh)

BLACKSITE is designed to conform to Section 508 of the Rehabilitation Act (36 CFR Part 1194, 2018 Refresh), which aligns with WCAG 2.1 Level AA. Conformance is partial; formal testing with assistive technologies (NVDA, JAWS, VoiceOver) is in progress for Q2 2026.

The VPAT details Chapter 3 (Functional Performance Criteria), Chapter 5 (Software), and Chapter 6 (Support Documentation) conformance claims. View VPAT

{# ── 3. Privacy ───────────────────────────────────────────────────────────── #}

3. Privacy

  • CCPA/CPRA (California): Full data subject rights supported. BLACKSITE does not sell or share personal information.
  • GDPR (EU/EEA): Legal bases documented. Data Processing Agreements available on request for enterprise customers.
  • COPPA: Platform is for professional use only; not directed at persons under 18. No knowing collection of minors' data.
  • No data sales: BLACKSITE does not sell, rent, or share personal information with third parties for marketing purposes.

View Privacy Notice

{# ── 4. Security ──────────────────────────────────────────────────────────── #}

4. Security Controls

Database Encryption
SQLCipher AES-256 encryption at rest. The database encryption key is stored in the systemd service environment, never in code or configuration files.
Immutable Audit Log
All authentication, access, and modification events are written to an append-only audit log with user, timestamp, IP address, and action details.
Role-Based Access Control
Granular RBAC with defined roles (Admin, ISSM, ISSO, SCA, System Owner, PMO, Employee, Read-Only). Access to records is enforced at the application layer.
Authelia SSO
Authentication is delegated to Authelia, providing centralized single sign-on, multi-factor authentication support, and session management.
Encryption in Transit
All traffic is served over HTTPS (TLS 1.2+) via a Caddy reverse proxy with automatic certificate management.
Password Security
Passwords are hashed with bcrypt. Plaintext passwords are never stored or logged. Session tokens are server-side and cryptographically random.
