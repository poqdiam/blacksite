WCAG 2.1 Report

Table 1: Success Criteria, Level A

Criterion Conformance Level Remarks
1.1.1 Non-text Content Supports All images include descriptive alt attributes. Icon-only buttons include aria-label. Color-only status indicators include aria-label.
1.2.1 Audio-only and Video-only (Prerecorded) Not Applicable Product contains no audio or video content.
1.2.2 Captions (Prerecorded) Not Applicable No multimedia content present.
1.2.3 Audio Description or Media Alternative (Prerecorded) Not Applicable No multimedia content present.
1.3.1 Info and Relationships Supports Semantic HTML throughout. Tables include scope attributes. Form inputs associated with labels. ARIA roles applied to custom components.
1.3.2 Meaningful Sequence Supports Content is presented in a logical reading order in source markup.
1.3.3 Sensory Characteristics Supports Instructions do not rely solely on shape, size, color, or location.
1.4.1 Use of Color Supports Status indicators include both color and text labels or aria-label attributes.
1.4.2 Audio Control Not Applicable No auto-playing audio present.
2.1.1 Keyboard Partially Supports Primary navigation and most interactive controls are keyboard-accessible. Some complex AJAX interactions may have keyboard limitations. Full keyboard regression testing in progress.
2.1.2 No Keyboard Trap Supports Modal dialogs implement focus traps with Escape key dismissal and focus restoration.
2.1.4 Character Key Shortcuts Not Applicable No single-character key shortcuts implemented.
2.2.1 Timing Adjustable Partially Supports Session idle timeout enforced via Authelia (configurable). Application-layer timeout warning implemented. Users can extend sessions.
2.2.2 Pause, Stop, Hide Supports No auto-moving, auto-updating, or auto-scrolling content.
2.3.1 Three Flashes or Below Threshold Supports No flashing content present.
2.4.1 Bypass Blocks Supports Skip navigation link ("Skip to main content") is the first focusable element on every page.
2.4.2 Page Titled Supports All pages include a descriptive <title> with application name and page context.
2.4.3 Focus Order Supports Focus order follows a logical sequence. Modals move focus to first focusable element on open and restore on close.
2.4.4 Link Purpose (In Context) Supports All links include descriptive text or aria-label. Symbol-only links include aria-label.
2.5.1 Pointer Gestures Not Applicable No multi-point or path-based gestures required.
2.5.2 Pointer Cancellation Supports Click events on mouseup/standard click; no down-event-only actions.
2.5.3 Label in Name Supports Visible labels match or are contained within accessible names.
2.5.4 Motion Actuation Not Applicable No motion-activated functionality present.
3.1.1 Language of Page Supports lang="en" declared on the <html> element.
3.2.1 On Focus Supports No context changes occur on focus alone.
3.2.2 On Input Partially Supports Auto-submitting <select> elements are labeled to inform users of auto-apply behavior.
3.3.1 Error Identification Partially Supports Most form errors are displayed inline. Some API error states surface as toast messages only.
3.3.2 Labels or Instructions Supports All form inputs have associated labels. Required fields include aria-required="true".
4.1.1 Parsing Supports HTML is validated. No duplicate IDs on interactive elements. ARIA used per specification.
4.1.2 Name, Role, Value Supports All interactive elements have accessible names, roles, and states. Custom components use appropriate ARIA.

Table 2: Success Criteria, Level AA

Criterion Conformance Level Remarks
1.2.4 Captions (Live) Not Applicable No live multimedia content.
1.2.5 Audio Description (Prerecorded) Not Applicable No multimedia content.

1.3.5 Identify Input Purpose Partially Supports Standard profile fields (name, email) do not yet include autocomplete attributes. Planned Q2 2026.
1.4.3 Contrast (Minimum) Not Evaluated Dark theme designed for high contrast. Formal contrast ratio measurement in progress (Q2 2026).
1.4.4 Resize Text Supports Interface uses relative units (em, rem). Text resizes to 200% without loss of content.
1.4.5 Images of Text Supports No images of text used. All text rendered as HTML.
1.4.10 Reflow Partially Supports Most pages reflow at 320px without horizontal scrolling. Complex data tables require horizontal scroll at narrow widths.
1.4.11 Non-text Contrast Not Evaluated UI components designed with contrast in mind. Formal measurement pending (Q2 2026).
1.4.12 Text Spacing Supports No loss of content or functionality when text spacing properties are overridden.
1.4.13 Content on Hover or Focus Supports Tooltips are dismissible and persistent. No hover content that obscures other content.
2.4.5 Multiple Ways Supports Navigation sidebar, page titles, and URL structure provide multiple means of locating pages.
2.4.6 Headings and Labels Supports Descriptive headings used throughout. Form labels are descriptive and associated with controls.
2.4.7 Focus Visible Partially Supports Browser default focus indicators present. Custom focus styles in some areas. Comprehensive styled focus rings in progress.
3.1.2 Language of Parts Not Applicable All content is in English. No foreign-language passages.
3.2.3 Consistent Navigation Supports Navigation sidebar and header are consistent across all pages.
3.2.4 Consistent Identification Supports Components with the same function are identified consistently throughout.
3.3.3 Error Suggestion Partially Supports Most validation errors include suggested corrections. Some API-level errors surface generic messages.
3.3.4 Error Prevention (Legal, Financial, Binding) Supports Destructive actions require confirmation. ATO decisions include explicit confirmation dialogs.
4.1.3 Status Messages Supports Dynamic content regions include aria-live="polite". Toast notifications are announced.
