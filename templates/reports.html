Authorization Package Reports

These reports are the standard deliverables submitted to an Authorizing Official (AO), Inspector General (IG), or certification body as part of an authorization or compliance package. Navigate to a specific system to download its package.

{# System Security Plan (SSP) #}
📄
System Security Plan (SSP) Agency
per-system
The primary authorization document. Describes the system boundary, its purpose, all applicable controls, and the implementation status of each. The SSP is the "source of truth" reviewed by the Authorizing Official before granting authorization.
Go to Systems → Select system → ⎙ Report
{# Security Assessment Report (SAR) #}
🔍
Security Assessment Report (SAR) Agency
per-system
Documents the results of the security control assessment. Produced by an independent assessor, it lists all controls tested, findings, and recommendations. Required before the Authorizing Official can make an authorization decision. Linked to uploaded assessments.
View Submissions Go to Systems →
{# ATO Authorization Decision #}
✅
ATO Status Report Agency
Authorization decisions & expiry tracker
Summary of authorization status across all systems in scope — Authorized, In Progress, Not Authorized, and Expired ATOs. Includes authorization dates and upcoming expirations. Used by compliance teams for continuous authorization monitoring.

RMF Lifecycle Summary Agency
org-wide
Cross-system view of the 7-step RMF lifecycle (Prepare → Categorize → Select → Implement → Assess → Authorize → Monitor). Shows which systems are at each stage, who owns each step, and target completion dates. Required for continuous authorization programs (cATO).
