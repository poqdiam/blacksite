ATO Package Reports

These reports are the standard deliverables submitted to an Authorizing Official (AO) or Inspector General (IG) as part of the Authorization to Operate (ATO) package. Navigate to a specific system to download its package.

{# System Security Plan (SSP) #}
📄
System Security Plan (SSP) Agency
NIST SP 800-18 · per-system
The primary ATO document. Describes the system boundary, its purpose, all applicable NIST 800-53 controls, and the implementation status of each. The SSP is the "source of truth" reviewed by the Authorizing Official before granting an ATO.
Go to Systems → Select system → ⎙ Report
{# Security Assessment Report (SAR) #}
🔍
Security Assessment Report (SAR) Agency
NIST SP 800-53A · per-system
Documents the results of the security control assessment (SCA). Produced by an independent Security Control Assessor (SCA), it lists all controls tested, findings, and recommendations. Required before the AO can make an authorization decision. Linked to uploaded assessments.
View Submissions Go to Systems →
{# ATO Authorization Decision #}
✅
ATO Status Report Agency
Authorization decisions & expiry tracker
Summary of authorization status across all systems in scope — Authorized, In Progress, Not Authorized, and Expired ATOs. Includes authorization dates and upcoming expirations. Used by compliance teams for continuous authorization monitoring.

{# RMF Lifecycle Summary #}
🔄
RMF Lifecycle Summary Agency
NIST SP 800-37 Rev 2 · org-wide
Cross-system view of the 7-step RMF lifecycle (Prepare → Categorize → Select → Implement → Assess → Authorize → Monitor). Shows which systems are at each stage, who owns each step, and target completion dates. Required for continuous authorization programs (cATO).
