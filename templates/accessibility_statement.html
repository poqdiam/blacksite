Accessibility Statement

Last assessed: 2026-03-09

{# ── Conformance status ──────────────────────────────────────────────────── #}

Conformance Status

Partially conformant with WCAG 2.1 Level AA. Partially conformant means that some parts of the content do not fully conform to the accessibility standard.

{# ── What we've done ─────────────────────────────────────────────────────── #}

Accessibility Improvements Completed

  • Skip navigation link — a "Skip to main content" link is present at the top of every page.
  • ARIA labels — interactive controls (buttons, inputs, icons) carry descriptive aria-label or aria-labelledby attributes.
  • Modal semantics — dialog elements use role="dialog", aria-modal="true", and focus management (trap + restore on close).
  • Table scope attributes — data tables include scope="col" and scope="row" on header cells.
  • Required field indicators — required form fields are marked with aria-required="true" in addition to visual cues.
  • Color-independent status indicators — status information is conveyed by text or icon in addition to color.
  • Tab panel ARIA — tabbed interfaces use role="tablist", role="tab", role="tabpanel", and aria-selected.
  • Live regions — dynamic status updates use aria-live="polite" or aria-live="assertive" so screen readers announce changes.
  • Keyboard navigation — all interactive elements are reachable and operable via keyboard alone.
{# ── Known limitations ───────────────────────────────────────────────────── #}

Known Limitations

  • Contrast ratios — foreground/background contrast ratios have not been formally measured against the WCAG 2.1 AA threshold (4.5:1 for normal text). Some muted text elements may fall below this threshold.
  • Keyboard regression testing — systematic keyboard-only regression testing is in progress. Some recently added interactive components may not yet meet full keyboard operability requirements.
  • Complex tables on narrow viewports — data-dense comparison tables may require horizontal scrolling on small screens. Column headers may not remain visible when scrolling.
{# ── Technical information ────────────────────────────────────────────────── #}

Technical Information

Target standard: WCAG 2.1 Level AA

Testing browsers: Google Chrome (latest stable), Mozilla Firefox (latest stable)

Assessment approach: Self-evaluation by the development team using browser developer tools, manual keyboard testing, and automated linting (axe-core where available).

View our full VPAT (Voluntary Product Accessibility Template) for detailed criterion-by-criterion conformance claims under WCAG 2.1 and Section 508.
