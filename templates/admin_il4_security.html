Closing Open Gaps

GAP-1 — Encryption at Rest (SC-28)
Install libsqlcipher3 and set security.db_encryption: true. Restart BLACKSITE. A SQLCipher-encrypted database will be used on next startup. Existing data must be migrated using sqlcipher CLI before enabling in production.

GAP-2 — Immutable Audit Log (AU-2/AU-10)
Closed by design. The immutable_audit_log table is always created and the IL4 audit middleware writes append-only entries for all admin, export, and mutation events. Enable security.audit_all_requests: true to log every HTTP request.

GAP-3 — MFA Enforcement (IA-2(1))
Set security.require_mfa: true. BLACKSITE will then require the X-Authelia-Auth-Level: two_factor header on all requests; ensure Authelia is configured with a two_factor policy for the BLACKSITE domain.

GAP-4 — FIPS Session Crypto (SC-13)
Set security.fips_session_crypto: true and BLACKSITE_ENV=prod. This is a PROD-only guard — the check is bypassed in development to avoid blocking the server.
