menu_book Setup Guide

Keycloak

  1. Create a Client in your realm with
    Client type: OpenID Connect
  2. Set redirect_uri in the client's Valid Redirect URIs
  3. Set discovery_url:
    https://<host>/realms/<realm>/.well-known/openid-configuration
  4. Add groups mapper to include group membership in token
  5. Map Keycloak group names in role_map

Okta

  1. Create an OIDC Web Application
  2. Add Sign-in redirect URI matching your redirect_uri
  3. Set discovery_url:
    https://<domain>/.well-known/openid-configuration
  4. Add Groups claim in token settings
  5. Map Okta group names in role_map

Azure AD

  1. Register an app in Entra ID
  2. Add a Web redirect URI matching redirect_uri
  3. Set discovery_url:
    https://login.microsoftonline.com/<tenant>/v2.0/.well-known/openid-configuration
  4. Set role_claim: roles and configure app roles
  5. Map Azure role names in role_map
Authelia co-existence: When Authelia injects a Remote-User header, SSO is bypassed entirely — existing deployments are unaffected. SSO activates only when no Remote-User header is present (e.g. direct access without Caddy/Authelia in the path).
